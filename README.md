## What Is a Design Pattern?

A design pattern is a solution to a commonly occurring problem in software design. Patterns are not finished pieces of code — they are pre-made blueprints that can be customized to solve a particular design problem in your own codebase.

## What Does a Pattern Consist Of?

Most patterns are described formally so that they can be reproduced in many different contexts. A typical pattern description includes the following sections:

| Section | Description |
|---|---|
| **Intent** | Briefly describes both the problem and the solution. |
| **Motivation** | Further explains the problem and the solution the pattern makes possible. |
| **Structure** | Shows each part of the pattern and how the parts relate to one another, usually via a class diagram. |
| **Code Example** | Implements the pattern in a popular programming language to make the idea easier to grasp. |

## Why Learn Patterns?

Design patterns are a toolkit of tried-and-tested solutions to common problems in software design. Even if you never run into these exact problems, knowing patterns is still useful — they teach you how to solve all kinds of problems using the principles of object-oriented design.

## Classification of Patterns

All patterns can be categorized by their **intent**, or purpose. There are three main groups:

### Creational Patterns

Provide object creation mechanisms that increase the flexibility and reuse of existing code.

- Factory Method
- Abstract Factory
- Builder
- Prototype
- Singleton

### Structural Patterns

Explain how to assemble objects and classes into larger structures, while keeping these structures flexible and efficient.

- Adapter
- Bridge
- Composite
- Decorator
- Facade
- Flyweight
- Proxy

### Behavioral Patterns

Take care of effective communication and the assignment of responsibilities between objects.

- Chain of Responsibility
- Command
- Iterator
- Mediator
- Memento
- Observer
- State
- Strategy
- Template Method
- Visitor
